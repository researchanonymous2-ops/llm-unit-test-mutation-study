# Artifacts of the Empirical Study

This repository contains the artifacts generated during the empirical study. The artifacts are organized by target class, following a uniform directory structure pattern described below.

## General Repository Structure

Each target class has a dedicated folder inside `experiments/`, named according to the pattern:

**Example:** `experiments/exp-anagrams/`, `experiments/exp-area/`, `experiments/exp-fraction/`, etc.

## Description of Directories and Files

### `mutation-testing/`
Contains the artifacts generated after applying mutation testing to the target class.

#### `equivalent-mutants-candidates/`
Contains a CSV file presenting the surviving mutants identified in the mutation reports of the test suites, along with a mapping of possible candidate equivalent mutants.

#### `fixed-mutants/`
Contains the `metadata.json` and `pitHistory.bin` files, as well as the subfolders:

- **`mutation-operators-report/`** — CSV with the mutation operators applied to the target class using Pitest.
- **`report/`** — HTML, CSV, and XML files generated by Pitest after generating mutants in the target class.

#### `quantitative-mutants-analysis/`
Contains a CSV with the mutation score values per eligible test suite, along with other metrics that allow a quantitative analysis of mutation testing.

#### `suites-execution-results/`
Contains directories identified by the names of all test suites that participated in mutation testing. Inside each suite's directory, there are HTML, CSV, and XML files generated by Pitest after applying mutation testing.

> **Directory naming pattern:** `<ModelClassName>_<promptStrategy>_<suiteNumber>`
> **Example:** `AnagramsGrok_COC2Test`

### `prompts/`
Contains 5 `.txt` files with the instructions (prompts) provided to the LLMs for generating unit test suites, according to each prompt strategy:

| File               | Strategy     			 |
|--------------------|---------------------|
| `zero-shot.txt`    | Zero-Shot    		   |
| `one-shot.txt`     | One-Shot     			 |
| `few-shot.txt`     | Few-Shot     			 |
| `pot.txt`          | Program of Thoughts |
| `coc.txt`          | Chain of Code			 |

These files are consumed by the test generation script for each target class.

---

### `src/`
Directory that contains, in a mirrored structure (same package), the subfolders:

- **`main/`** — Contains the target class of the study.
- **`test/`** — Contains all test suites eligible for mutation testing. The ineligible suites are located in `test-suite-validation/non-eligible-suites/`. This directory also contains the base test suite, used only to activate mutation points in the target class (e.g., `AnagramsBaseTest`).

---

### `target/`
Directory where Maven deposits all artifacts generated during the project build. This is a temporary build output folder, automatically generated by Maven.

---

### `test-suite-validation/`
Directory responsible for validating test suites for compilation and execution.

#### `logs-suites/`
Contains failure logs organized into:

- **`compilation/failed/`** — Individual logs for each suite with compilation errors and a CSV consolidating all errors.
- **`execution/failed/`** — Individual logs for each suite with execution errors and a CSV consolidating all errors.

#### `non-eligible-suites/`
Contains test suites ineligible for mutation testing (`.java` files).

#### `suites-quantitative-report/`
Contains a CSV with the total number of suites generated per model and prompt strategy, as well as the quantities of eligible and ineligible suites.

---

### `pom.xml`
The Project Object Model (POM) is Maven's central configuration file. It defines everything Maven needs to know about the project, including:

- 📦 Dependencies
- ⚙️ Plugins
- 🏗️ Build process
- 📁 Project structure
- 📌 Project version
- 📋 Project metadata

## Files in the Repository Root

### `target-test-classes-and-methods.csv`
CSV file indicating the study's datasets, containing the analyzed target classes, the cyclomatic complexity of each class, and the target test methods.

## Scripts

The `scripts/` directory contains all scripts used to automate the steps of the empirical study. The scripts are numbered sequentially (01–10), indicating the **execution order** of the experimental pipeline, and are organized into three subfolders according to the experiment phase:

```
scripts/
├── create-experiment/
├── test-suites/
└── mutation-testing/
```

---

### `create-experiment/`

Initial phase of the pipeline. Creates the standard directory structure for a new target class.

| Script | Description |
|--------|-------------|
| `01-new-experiment.sh` | Creates the standard folder and subfolder structure for the experiment of a new target class |

---

### `test-suites/`
Phase for generating and validating unit test suites via LLMs.

| Script | Description |
|--------|-------------|
| `02-generate-test-suites.py` | Consumes the prompt files and triggers the LLMs to generate test suites for the target class |
| `03-validate-suites-tests.py` | Verifies the compilation and execution of the generated suites, separating eligible from ineligible ones |
| `04-generate-test-suites-failures-report.py` | Generates compilation and execution error log reports with deduplication |
| `05-generate-test-suites-quantitative-report.py` | Generates the quantitative report of suites per model and prompt strategy |

---

### `mutation-testing/`
Phase for applying mutation testing with Pitest.

| Script | Description |
|--------|-------------|
| `06-generate-target-class-mutants.sh` | Executes Pitest to generate mutants in the target class |
| `07-generate-target-class-mutation-operators-report.sh` | Generates the report of mutation operators applied to the target class |
| `08-run-test-suites-on-mutants.sh` | Executes the eligible test suites on the generated mutants |
| `09-generate-mutation-score-report.py` | Generates the report with mutation score values per test suite |
| `10-generate-surviving-mutants-report.py` | Generates the mapping of surviving mutants and candidate equivalents |
